VOID-Core v2.0: Полное руководство по разработке
VOID-Core — это гибридная среда, где код адаптируется под уровень угрозы. Вы пишете один проект, который одновременно является драйвером, логическим ботом и веб-интерфейсом.
1. Структура файлов и Слои (Layers)
Проект делится на три расширения. Компилятор определяет права доступа (Privileges) на основе расширения файла и внутреннего маркера.
Файл	Слой	Маркер	Описание
.ctos	[SYS]	//---- [SYS] ----/	Броня. Работа с BIOS, регистрами и Ghost-RAM.
.vc	[LOGIC]	//---- [LOGIC] ----/	Мозг. Асинхронная логика, SQL, Playwright, API.
.web	[VIEW]	//---- [VIEW] ----/	Лицо. Интерфейс управления (HTML/CSS/JS).
2. Универсальный тип cell (Ячейки)
Забудьте про int или string. В VOID-Core всё есть ячейка. Её поведение в памяти зависит от модификатора.

    cell:vocal — Обычные данные (как в Python). Используются в слое [LOGIC]. Автоматически очищаются сборщиком мусора.
    cell:ghost — Шифрованные данные. Расшифровываются только внутри CPU. Идеально для паролей и ключей. Невидимы для дампов памяти.
    cell:fwr — BIOS-Lock. Данные, привязанные к аппаратной подписи прошивки. Если BIOS изменен — ячейка превращается в мусор.
    cell:raw — Прямой указатель на адрес (как в C++). Используется для максимальной скорости в слое [SYS].

3. Ключевые операторы

    &- (Bridge): Оператор «прокола». Позволяет вызвать защищенную функцию из [SYS] внутри обычного приложения [LOGIC].
        Пример: &- HardwareGuardian.wipe();
    move: Передача владения (Rust-style).
        Пример: move secret -> vault; (После этого secret физически стирается из памяти нулями).
    decide / weight: Твоя замена if/switch. Позволяет системе оценивать приоритетность выполнения задачи.

4. Как писать код (Пример структуры)
Системный уровень (security.ctos)
Здесь мы прописываем «железные» правила.
cpp

//---- [SYS] ----/
node Shield {
    cell:fwr MASTER_ID = "VOID_AF_2026"; // Привязка к прошивке

    pub fn verify() -> bool {
        // Проверка: мы на своем железе?
        if (sys.hwid() != MASTER_ID) {
            sys.trigger_fake_panic("0xC0000005"); // Имитируем краш Windows
            return false;
        }
        return true;
    }
}

Use code with caution.

5. Протоколы безопасности (Золотые правила)

    Скрытность: Всегда используй sys.unlink_peb() в начале Windows-приложения, чтобы исчезнуть из списка процессов.
    Паранойя: Если твоя программа запущена под отладчиком (IDA/Ghidra), рантайм void.run автоматически должен вызывать sys.self_destruct().
    Гигиена: По завершении работы всегда вызывай mem.physical_shred(). Данные не должны оставаться в RAM после закрытия программы.
    Сетевой пробив: Используй net.raw_inject(packet, "L2_BYPASS"), чтобы пакеты уходили в сеть в обход системного VPN или Proxy.

6. Компиляция и запуск

    Сборка: void_win.exe builder.vc — Мастер-сборщик соберет все слои в один файл.
    Результат: Ты получишь файл void.run.
    Запуск: voidrun void.run (или просто запуск через лоадер).
