README.md для версии v2.0 (Next-Gen Solution Language) зафиксирована финальная архитектура всей экосистемы. Это не просто описание, а технический паспорт проекта.
Вот краткое содержание того, что там прописано:
1. Архитектурный манифест

    Универсальность: Поддержка Windows (x64) и Android 10-15 (ARM64).
    Runtime: Самоадаптирующийся движок void.run, который сам управляет ядрами процессора (BIG.LITTLE) и скрытностью.
    Security: Многоуровневый щит — от проверки подписи BIOS до блокировки BadUSB на уровне прерываний.

2. Структура Репозитория (Назначение папок)

    📁 View/ — Интерфейсы управления (.web), включая Rev-Engine для перехвата данных.
    📄 kernel.ctos / memory.ctos — Низкоуровневое ядро, управляющее Ghost-RAM (зашифрованной памятью).
    📄 firmware.ctos — Модуль v2.0 для защиты пред-загрузочного состояния.
    📄 sql.core — Движок данных, пишущий в скрытые (битые) сектора диска через HiddenVault.
    📄 compiler.core — Твой мульти-режимный транслятор.

3. Протокол запуска (Инструкция)

    Настройка: Обязательная корректировка MASTER_HWID под твой Dell или Android.
    Сборка: Команда для void_win.exe builder.vc, которая сплавляет слои в один бинарник.
    Итог: Получение void.run, готового к работе в режиме полной невидимости.

4. Дисклеймер (Предупреждение)

    Указано, что функции типа net.raw_inject и sys.fake_panic — это мощные инструменты, и их использование на чужих устройствах приведет к потере доступа.
